Chapter 4: Setup vs. Loop (The Logic Flow)

If you are coming from Arduino, you are used to void setup() and void loop(). In Python, we use global space and while loops to do the same thing.

Arduino Concept Python Equivalent Purpose
void setup() { ... } Top of Script Run once at the start (define pins, variables, or constants).
void loop() { ... } while True: Repeat the code forever (checking sensors, buttons, or user input).

Example: The Blink Logic

# --- SETUP SECTION (Runs once) ---
import time
led_status = False

# --- LOOP SECTION (Runs forever) ---
while True:
    led_status = not led_status # Toggle LED
    print("LED is:", led_status)
    time.sleep(1) # Wait 1 second

Where do I write my functions?

Always define your functions above where you call them. Usually, this goes in the "Setup" area before your main loop starts.

Chapter 6: Loops (Repeat Actions)

Loops save you from writing the same code over and over. Python has two main types: for and while.

The 'For' Loop

Use this when you know how many times you want to repeat something (e.g., through a list or a range of numbers).

for i in range(5):
    print("Number:", i) # Prints 0, 1, 2, 3, 4

tools = ["Linux", "Python", "Arduino"]
for item in tools:
    print("Learning:", item)

The 'While' Loop

Use this when you want to repeat until a condition changes (e.g., until a user types "quit").

count = 10
while count > 0:
    print("Countdown:", count)
    count -= 1 # Subtract 1 each time
print("Blast off!")

Loop Controls

Command What it does Use Case
break Stops the loop instantly. Exit a while True: loop when a button is pressed.
continue Skips the rest of the current turn and restarts the loop. Skip "even" numbers in a list.
range() Creates a sequence of numbers. range(start, stop, step)

Chapter 2: Creating Your Own Tools (def)

A Function is a block of code which only runs when it is called. You can pass data, known as parameters, into a function.

The 'def' Keyword

To create a function, use def followed by the name and parentheses. You write these in your "Setup" area.

def greet_user(name):
    print("Hello, " + name + "!")

# Now you can use your new command anywhere:
greet_user("Mellor")
greet_user("Admin")

Inputs & Outputs (Arguments & Return)

Concept How to Write It Purpose
Arguments def func(x, y): The variables you "send" into the function to be processed.
return return result The value the function "gives back" to the main script.
*args def func(*args): Used when you don't know how many arguments will be sent.

Example: A Math Tool

def calculate_area(width, height):
    area = width * height
    return area

# Using the return value
my_room = calculate_area(10, 12)
print(f"Room size: {my_room} sq ft")

Where to write them? Always define your def blocks at the very top of your script, right after your imports. If you try to call a function before you define it, Python will crash!

Chapter 9: File Handling (Read & Write)

In Python, you can open, read, write, and delete files. This is how you save user data, logs, or configuration settings.

The 'open()' Function

The open() function takes two arguments: the filename and the mode.

Mode Description Use Case
"r" Read: Opens a file for reading (Default). Reading a list of passwords or settings.
"w" Write: Overwrites the file with new data. Saving a fresh high score.
"a" Append: Adds new data to the end of the file. Adding a new line to a log file.

The Modern Way: 'with open'

Using with is the "great" way to handle files because it automatically closes the file for you, preventing memory leaks.

# Writing to a file
with open("test.txt", "w") as f:
    f.write("Hello Mellor!\nWelcome to Python Mastery.")

# Reading from a file
with open("test.txt", "r") as f:
    content = f.read()
    print(content)

Useful File Commands

Command Effect
f.read() Reads the entire file as one long string.
f.readlines() Reads the file and puts each line into a List.
os.remove("file.txt") Deletes a file (requires import os).

Chapter 10: Bulletproofing Your Code (Try/Except)

In Python, an error that happens during execution is called an Exception. If you don't "catch" it, your program will stop immediately.

The 'Try...Except' Block

Think of this like a safety net for your code. You try to run a block of code, and if an error happens, the except block catches it.

try:
    number = int(input("Enter a number: "))
    result = 10 / number
    print(result)
except ValueError:
    print("Error: You didn't enter a valid number!")
except ZeroDivisionError:
    print("Error: You cannot divide by zero!")

Handling Specific Errors

Error Type When it happens
FileNotFoundError When you try to open a file that doesn't exist.
IndexError When you try to access a list item that isn't there (e.g., item 10 in a list of 5).
KeyError When you look for a key in a Dictionary that hasn't been created.
Exception The "Catch-All"—it catches any error that happens.

The 'Finally' Clause

The finally block will run no matter what, whether an error happened or not. This is great for closing files or stopping a motor on an Arduino.

try:
    f = open("data.txt")
except:
    print("Something went wrong.")
finally:
    print("This always runs (Good for cleanup!)")

Chapter 11: Blueprints & Objects (OOP)

Imagine you are building a game like Minecraft. You don't want to write code for every single block. Instead, you create a Class (the blueprint) for a "Block" and then create thousands of Objects from it.

The 'class' Keyword

A class contains Attributes (variables like color or health) and Methods (functions the object can do).

class Player:
    def __init__(self, name, health):
        self.name = name # Attribute
        self.health = health

    def take_damage(self, amount):
        self.health -= amount
        print(f"{self.name} now has {self.health} HP")

# Creating Objects from the Class
p1 = Player("Mellor", 100)
p1.take_damage(20)

OOP Terminology

Term What it is Analogy
Class The Blueprint. The drawing of a car.
Object The Instance. The actual car you drive.
__init__ The Constructor. Setting the color and engine when the car is built.
self The current object. The specific car you are sitting in right now.

You can create a "Child" class that copies everything from a "Parent" class but adds its own twist.

class HackerPlayer(Player): # Inherits from Player

        print("Scanning network...")

# This player has health AND the ability to scan!

Chapter 12: The Python Shortcut (List Comprehensions)

List comprehensions provide a concise way to create lists. Common applications are to make new lists where each element is the result of some operations applied to each member of another sequence.

The Old Way vs. The Pro Way

Imagine you want to create a list of squares for numbers 0 to 9.

# The Old Way (4 lines)
squares = []
for x in range(10):
    squares.append(x**2)

# The Pro Way (1 line!)
squares = [x**2 for x in range(10)]

Adding Logic (If Statements)

You can even add an if inside the shortcut to filter data instantly.

# Only keep even numbers from a list
numbers = [1, 2, 3, 4, 5, 6, 7, 8]
evens = [x for x in numbers if x % 2 == 0]
# Result: [2, 4, 6, 8]

When to use this?

Scenario Benefit
Data Filtering Quickly removing unwanted items from a list of logs or IPs.
Transformation Converting a list of strings to integers in one go.
Performance List comprehensions are actually slightly faster than standard loops in Python.

Warning: Don't make them too long. If a list comprehension is longer than one line, it’s better to use a normal for loop so your code stays readable!

Chapter 13: Power & Efficiency (Decorators & Generators)

These two concepts separate the beginners from the experts. Generators save memory, and Decorators modify how functions work.

1. Generators (yield)

A normal function returns a full list and stops. A Generator "yields" one item at a time. This is vital if you are scanning millions of IP addresses or reading a 10GB file.

def count_to_million():
    n = 1
    while n <= 1000000:
        yield n # Pauses here and gives the value
        n += 1

# This doesn't store 1,000,000 numbers in RAM!
for num in count_to_million():
    print(num)

2. Decorators (@)

Decorators allow you to "wrap" a function in another function to add extra code before or after it runs. In cybersecurity, this is often used for Logging or Authentication.

def admin_only(func):
    def wrapper():
        print("Checking permissions...")
        func()
    return wrapper

@admin_only
def delete_database():
    print("Database Deleted.")

Comparison Table

Concept The "Trick" The Benefit
Generators Uses yield instead of return. Uses 0% extra RAM even for massive datasets.
Decorators Uses @function_name above a definition. Clean code; you don't have to rewrite logic for every function.

Chapter 14: Talking to the Web (APIs & Scraping)

Modern Python is often used to pull data from the internet. You can use APIs (official data doors) or Web Scraping (extracting data from a website's HTML).

1. Using APIs with 'requests'

An API (Application Programming Interface) is like a menu at a restaurant. You ask for a specific "dish" (data), and the server brings it to you in JSON format.

import requests

# Getting data from a public API
response = requests.get("https://api.example.com/data")
data = response.json()
print(data["value"])

2. Web Scraping with 'BeautifulSoup'

When a site doesn't have an API, you "scrape" it. This means your script reads the HTML code of the page and finds the info you want.

from bs4 import BeautifulSoup
import requests

html = requests.get("https://google.com").text
soup = BeautifulSoup(html, "html.parser")
print(soup.title.text) # Prints the website title

The HTTP Status Codes

When you talk to the web, the server sends back a code to tell you what happened:

Code Meaning What it means for you
200 OK Success! You got the data.
404 Not Found The URL or page doesn't exist.
403 Forbidden The website is blocking your script (Anti-Scraping).
500 Server Error The website's computer crashed.

Chapter 15: The Final Level (Cybersecurity)

Python is the #1 language for cybersecurity because of its powerful libraries that can "sniff" network traffic, scan ports, and automate pentesting tasks.

1. Socket Programming (Port Scanning)

A Socket is an internal endpoint for sending or receiving data. You can use it to check if a "door" (port) is open on a server.

import socket

def scan_port(ip, port):
    s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    s.settimeout(1)
    if s.connect_ex((ip, port)) == 0:
        print(f"Port {port} is OPEN")
    s.close()

scan_port("127.0.0.1", 80)

2. Automating Linux Commands (Subprocess)

Since you use Linux Mint and Arch, you can use Python to run terminal commands (like nmap or ifconfig) and read the output.

import subprocess

# Running a terminal command via Python
result = subprocess.check_output(["ls", "-l"])
print(result.decode())
